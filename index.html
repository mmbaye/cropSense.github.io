00Getting Started › End-to-End Workflow

This chapter walks you through a complete analysis from opening the app to obtaining predictions with uncertainty estimates. Follow these steps in order for any new crop or trait.

CropSense › Spectra Tab (Sorghum, SNV preprocessing)
Fig. 1 › The Spectra tab showing 242 sorghum SNV-preprocessed spectra coloured by type_of_material. The sidebar controls preprocessing method and display options. The four metric cards at the bottom summarise the dataset.
💡 About the "type_of_material" dropdown

type_of_material refers to the plant material category, NOT the NIR instrument. For sorghum it contains: ERS Tall, ERS Short, EWS Tall, EWS Short (Early Red/White Sorghum × height class). It is useful in the Spectra and PCA tabs to check whether different material types cluster separately › which would suggest you need separate calibration models. If all types overlap spectrally, a single model is sufficient.

03Spectral Preprocessing › Methods & Equations

Raw NIR spectra contain physical artefacts › particle size variation, baseline drift, path-length differences › that mask the chemical signal. Preprocessing removes these before modelling.

3.1 › Standard Normal Variate (SNV)

SNV centres and scales each spectrum individually. It removes both additive (offset) and multiplicative (scaling) scatter effects without needing a reference spectrum.

For sample i with absorbance vector $\mathbf{x}_i = [x_{i1}, x_{i2}, \ldots, x_{ip}]$:

SNV transformation $$x_{ij}^{\text{SNV}} = \frac{x_{ij} - \bar{x}_i}{\sigma_i}$$

where $\bar{x}_i = \frac{1}{p}\sum_j x_{ij}$ and $\sigma_i = \sqrt{\frac{1}{p-1}\sum_j (x_{ij} - \bar{x}_i)^2}$, with $p$ = number of wavelengths.

✅ When to use SNV
  • First choice for FOSS NIR data (400–2500 nm)
  • Effective when samples differ in particle size or packing density
  • Sample-independent › applies identically to calibration and new samples

3.2 › Multiplicative Scatter Correction (MSC)

MSC corrects scatter by regressing each spectrum against the mean spectrum. More powerful than SNV when scatter is truly multiplicative, but depends on the calibration set.

Step 1 › reference spectrum $$\bar{x}_j = \frac{1}{n}\sum_{i=1}^n x_{ij}$$
Step 2 › OLS regression per sample $$x_{ij} = \hat{b}_{0i} + \hat{b}_{1i}\,\bar{x}_j + \varepsilon_{ij}$$
Step 3 › MSC-corrected spectrum $$x_{ij}^{\text{MSC}} = \frac{x_{ij} - \hat{b}_{0i}}{\hat{b}_{1i}}$$

3.3 › Detrend

Removes a linear baseline by fitting and subtracting a straight line from each spectrum:

Detrend $$x_{ij}^{\text{det}} = x_{ij} - (\hat{\beta}_{0i} + \hat{\beta}_{1i} \cdot j)$$

3.4 › Savitzky-Golay (SG) Derivatives

SG derivatives apply polynomial smoothing and simultaneously compute derivatives. They eliminate baseline effects while preserving spectral shape.

SG convolution filter $$x_i^{\text{SG}}(j) = \sum_{k=-(w-1)/2}^{+(w-1)/2} c_k \cdot x_i(j+k)$$

where $c_k$ are precomputed SG coefficients for window $w$, polynomial degree $p$, and derivative order $m$.

MethodRemovesEnhancesRisk
1st Derivative (m=1)Additive baselinePeak positions & shouldersNoise amplification
2nd Derivative (m=2)Linear + curved baselinePeak resolutionStrong noise (use large w)
💡 Recommended SG settings
  • Window size w = 11 (default, must be odd)
  • Polynomial degree p = 2 (default)
  • Larger w → smoother but lower spectral resolution

04Principal Component Analysis (PCA)

PCA transforms the high-dimensional spectral matrix (4200 wavelengths) into uncorrelated axes › Principal Components (PCs) › that capture maximum variance. It is unsupervised: it does not use trait values.

4.1 › Mathematical background

Given the mean-centred spectral matrix $\tilde{X}$ $[n \times p]$ (rows = samples, columns = wavelengths):

Covariance matrix $$\mathbf{S} = \frac{1}{n-1}\tilde{X}^\top \tilde{X} \quad [p \times p]$$
Eigenvalue decomposition $$\mathbf{S} = \mathbf{V} \boldsymbol{\Lambda} \mathbf{V}^\top$$

where $\mathbf{V}$ = eigenvectors (loadings), $\boldsymbol{\Lambda}$ = diagonal matrix of eigenvalues $\lambda_1 \geq \lambda_2 \geq \cdots$

Score matrix (projection) $$\mathbf{T} = \tilde{X} \cdot \mathbf{V}_k \quad [n \times k]$$

4.2 › Explained variance

Variance explained by PC k $$\text{Var}_k\,(\%) = \frac{\lambda_k}{\sum_i \lambda_i} \times 100$$
💡 Interpretation rule of thumb

For FOSS NIR data, 2–5 PCs typically explain 95%+ of spectral variance. The "elbow" in the scree plot marks where additional PCs give diminishing returns.

05PLS-R Calibration › Theory, Equations & Metrics

Partial Least Squares Regression (PLS-R) is the gold-standard method for NIR calibration. Unlike PCA, it simultaneously maximises the covariance between spectra $X$ and trait $Y$.

5.1 › Why not ordinary regression?

OLS fails with NIR data because: (1) $p \gg n$ (4200 predictors, ~250 samples), and (2) severe multicollinearity › adjacent wavelengths are almost perfectly correlated. PLS-R solves both problems by projecting onto a small number of latent variables (LVs).

5.2 › The PLS-R model

Simultaneous decomposition $$X = T P^\top + E_X \qquad Y = T Q^\top + E_Y$$
Prediction equation $$\hat{Y} = X\mathbf{B} = X\mathbf{W}(P^\top \mathbf{W})^{-1}Q^\top$$
T [n×k]
Score matrix › latent variable coordinates for each sample
P [p×k]
X loadings › spectral structure captured by each LV
Q [1×k]
Y loadings › how each LV relates to the trait
W [p×k]
X weights › wavelength importance for each LV
B [p×1]
Regression coefficients › one per wavelength

5.3 › Cross-validation and LV selection

CropSense uses 10-fold CV to select the optimal LV. The dataset is partitioned into 10 folds; each fold is predicted by a model trained on the remaining 9:

Optimal number of LVs $$\text{LV}_{\text{opt}} = \underset{k}{\arg\min}\;\text{RMSECV}(k)$$
🚨 Never report only calibration metrics

R² Cal and RMSEC always improve as LVs increase › even with random noise (overfitting). The red dashed line on the RMSECV plot marks LV_opt. Only R² CV and RMSECV are honest metrics.

5.4 › Statistical metrics

R² › Coefficient of Determination

R² (general) $$R^2 = 1 - \frac{SS_{\text{res}}}{SS_{\text{tot}}} = 1 - \frac{\sum_i(y_i - \hat{y}_i)^2}{\sum_i(y_i - \bar{y})^2}$$
R² CV $$R^2_{\text{CV}} = 1 - \frac{\sum_i(y_i - \hat{y}_i^{\text{CV}})^2}{\sum_i(y_i - \bar{y})^2}$$

RMSEC & RMSECV

Calibration error (optimistic) $$\text{RMSEC} = \sqrt{\frac{1}{n}\sum_{i=1}^n (y_i - \hat{y}_i^{\text{cal}})^2}$$
Cross-validation error (honest) $$\text{RMSECV} = \sqrt{\frac{1}{n}\sum_{i=1}^n (y_i - \hat{y}_i^{\text{CV}})^2}$$

RPD › Ratio of Performance to Deviation

RPD $$\text{RPD} = \frac{\text{SD}(y)}{\text{RMSECV}} = \frac{\sqrt{\frac{1}{n-1}\sum_i(y_i-\bar{y})^2}}{\text{RMSECV}}$$
> 4.0
Excellent › can replace wet lab✅
3.0–4.0
Very good › quality control✅
2.5–3.0

RPIQ › Ratio of Performance to InterQuartile range

More robust than RPD for skewed trait distributions (not normally distributed):

RPIQ (Bellon-Maurel et al. 2010) $$\text{RPIQ} = \frac{Q_{75} - Q_{25}}{\text{RMSECV}}$$

Bias

Mean signed error $$\text{Bias} = \frac{1}{n}\sum_{i=1}^n (y_i - \hat{y}_i)$$

6.3 › Prediction clamping

Predictions outside the calibration range are clamped to prevent unrealistic values:

Clamping (15% buffer) $$\hat{y}_{\text{clamped}} = \text{clamp}\!\left(\hat{y},\; y_{\min} - 0.15 \cdot \text{IQ},\; y_{\max} + 0.15 \cdot \text{IQ}\right)$$
⚠ Many predictions at the boundary?

This indicates the new spectra lie outside the calibration space › the model is extrapolating. Collect more calibration samples covering the full trait range.

07Gaussian Process Regression + Uncertainty

Gaussian Process (GP) Regression is a Bayesian non-parametric method that produces both a prediction and a probability distribution over possible values › unlike PLS-R which returns only a single number.

📌 Why uncertainty matters

A PLS-R model says: "protein = 11.2%". A GP says: "protein = 11.2% ± 2.1% (95% CI)". This distinction drives the wet lab alert system: only send uncertain samples back to wet lab, not all of them.

7.1 › The GP prior and posterior

GP prior $$f(\mathbf{x}) \sim \mathcal{GP}\!\left(m(\mathbf{x}),\; k(\mathbf{x}, \mathbf{x}')\right)$$
GP posterior prediction $$p(y^* \mid \mathbf{x}^*, X, \mathbf{y}) = \mathcal{N}(\mu^*, {\sigma^*}^2)$$

7.2 › The RBF kernel

CropSense uses the Radial Basis Function (Squared Exponential) kernel:

RBF kernel $$k(\mathbf{x}, \mathbf{x}') = \sigma^2 \exp\!\left(-\frac{\|\mathbf{x} - \mathbf{x}'\|^2}{2\ell^2}\right)$$
ℓ
Length-scale › controls smoothness. Large ℓ: predictions near the mean. Small ℓ: only nearby training points influence predictions.
σ²
Signal variance › automatically set to var(y).
σ²_n
Noise variance › models measurement error in y. Optimized automatically.

7.3 › Dimensionality reduction

Applying GP on 4200 wavelengths is intractable. CropSense first reduces X to PLS scores:

Dimension reduction $$X\;[n \times 4200] \;\xrightarrow{\text{PLS}}\; T\;[n \times 8]$$
Standardisation of scores $$T_{\text{norm}} = \frac{T - \mu_T}{\sigma_T}$$

7.4 › Hyperparameter optimisation

Parameters ℓ and σ²_n are found by maximising the log marginal likelihood:

Log marginal likelihood (Rasmussen & Williams 2006) $$\log p(\mathbf{y} \mid X, \boldsymbol{\theta}) = -\frac{1}{2}\mathbf{y}^\top K^{-1}\mathbf{y} - \frac{1}{2}\log|K| - \frac{n}{2}\log(2\pi)$$

where $K = \sigma^2 K_{\text{RBF}} + \sigma^2_n I$ and optimisation uses L-BFGS-B in log-parameter space.

7.5 › Exact LOO predictions

CropSense computes exact Leave-One-Out predictions using the closed-form result from Rasmussen & Williams (2006), §5.4.2. All n LOO predictions require only ONE matrix inversion (O(n³)):

Full covariance matrix $$K = \sigma^2 K_{\text{RBF}} + \sigma^2_n I$$
Precompute alpha (Cholesky) $$\boldsymbol{\alpha} = K^{-1}\mathbf{y} \quad \text{via Cholesky decomposition}$$
LOO predictive mean $$\mu_{\text{LOO},i} = y_i - \frac{\alpha_i}{[K^{-1}]_{ii}}$$
LOO predictive variance $$\sigma^2_{\text{LOO},i} = \frac{1}{[K^{-1}]_{ii}}$$

These are exact results › not approximations. They are equivalent to refitting the GP n times but require only one factorisation.

7.6 › Confidence intervals and alerts

95% confidence interval for sample i $$\text{CI}_{95\%,i} = \left[\mu_{\text{LOO},i} - 1.96\,\sigma_{\text{LOO},i},\;\; \mu_{\text{LOO},i} + 1.96\,\sigma_{\text{LOO},i}\right]$$
Wet lab alert condition $$\text{alert}_i = \mathbf{1}\!\left[1.96\,\sigma_{\text{LOO},i} > \tau\right]$$

where τ is the threshold slider (half-width). The ci95_width column = $2 \times 1.96 \times \sigma_{\text{LOO},i}$ (full width).
